# anki

A bsrc component that wires the user's Anki spaced-repetition workflow
into Claude Code and Codex.

Two surfaces:

- **Anki MCP server addon** ([`124672614`][mcp-addon], listens on
  `http://127.0.0.1:3141/`). The agent's authoring channel ‚Äî direct
  tools to add/update notes, list decks, manage tags, trigger sync.
- **Cards git repo** ‚Äî version-controlled markdown snapshot of the
  collection. Generated by `ank_dump.py` (talks to the MCP),
  committed/pushed by `ank_sync.sh`.

The collection itself syncs across devices via AnkiWeb (Anki's free
service); bsrc never touches AnkiWeb credentials.

`ank_dump.py` is a single-file Python script with no dependencies
(stdlib only). Run via uv (PEP 723 header) or any Python 3.11+.

## Requirements

- macOS (rest of bsrc)
- A separate GitHub repo for your cards markdown (created by you)
- An AnkiWeb account; sign in once per device

There is no AnkiConnect, no AnkiOps, no Python tool to install. The
only external piece beyond Anki itself is the MCP server addon, which
runs inside Anki.

## Install

```bash
./install.sh --cards-dir <path>
# or from the repo root:
./setup.sh --with-mcps
```

The installer:
1. Installs Anki desktop via `brew install --cask anki`.
2. Clones `$CARDS_REPO_URL` to `--cards-dir` (default
   `$HOME/Developer/personalprojects/anki-cards`). Skips if already a
   git repo. Errors if the path exists with non-git content.
3. Persists `ANKI_CARDS_DIR=<path>` to `~/.config/bsrc/anki.env`.
4. Symlinks `skill/` into `~/.claude/skills/anki` and
   `~/.codex/skills/anki`.
5. With `--with-mcps`: registers the Anki MCP entry for Claude Code.
   Codex picks up the same entry from `bsrc/codex/mcp-servers.toml`.
6. Prints a one-time per-device setup checklist.

## One-time per-device setup (manual, after install)

1. **Install the Anki MCP addon.** Open Anki: Tools ‚Üí Add-ons ‚Üí
   Get Add-ons ‚Üí enter `124672614`. Restart Anki.

2. **Sign in to AnkiWeb** inside Anki (Sync button or Tools ‚Üí
   Preferences ‚Üí Network).

3. **Verify.**
   ```bash
   bash ~/.claude/skills/anki/scripts/ank_doctor.sh
   ```

## Daily workflow

- Author cards via Claude/Codex. They use the Anki MCP tools
  (`add_note`, `add_notes`, etc.). Cards land in your collection
  immediately.
- Periodically snapshot to git:
  ```bash
  bash ~/.claude/skills/anki/scripts/ank_sync.sh           # commit only
  bash ~/.claude/skills/anki/scripts/ank_sync.sh --remote  # + git push
  ```
- AnkiWeb syncs cards + review state to phone/web/other Macs
  automatically (when you press Sync in Anki, or per its auto-sync
  setting).

## Cards-dir discovery

Skill scripts find the cards dir by:

1. `$ANKI_CARDS_DIR` env var
2. `~/.config/bsrc/anki.env` (written at install time)
3. Walking the cwd upward for the `.anki-cards` marker
4. Hard fail with init hint

No built-in default. The path is set at install time.

## Markdown format

One file per deck. Q is the heading, A is the body, identity goes
inline:

```markdown
# Torch
*19 cards ¬∑ regenerated 2026-05-08 from Anki via MCP ¬∑ do not hand-edit*

## What are the three core components of the PyTorch library?
*Basic ¬∑ id 3a2a9bf23def*

1. The `Tensor` Library...
2. The automatic-differentiation engine...
3. The Deep learning library...

---

## What is the difference between `.reshape` and `.view` in PyTorch?
*Basic ¬∑ id 0c6eb4cd58c1*

Both methods are used to reshape...
```

The deck files are a **snapshot**, not a write surface. Editing them
by hand is silently undone on the next `ank_sync.sh` run. Author via
the agent + MCP, or via Anki's UI.

## Why this shape

Earlier iterations bundled AnkiOps + AnkiConnect for bidirectional
markdown‚ÜîAnki sync. The agent never needed bidirectional ‚Äî it writes
through the MCP, which talks straight to the collection. AnkiOps's
unique value (push markdown edits back to Anki) was unused, and it
brought a label-namespace dance, an upstream `html-to-markdown` bug
pin, the `.ankiops.db` per-device state file, and the AnkiConnect
addon as a transitive dependency. Cutting all of that left the same
outcome with one small Python script (`ank_dump.py`) reading from the
MCP we already have.

## Slice 2 deferrals

- `knowledge-smith` ‚Üí cards bridge: when the user has just read a
  paper in their KS vault, agent offers to extract cards into the
  matching Anki deck (tag `ks:<slug>`, `paper:<id>`). Documentation-
  only change to both `SKILL.md`s.
- Quality pass over the existing collection: dupe detection, weak
  card surfacing, controlled tag vocab.
- Daily-review reminder hook tied into peon-ping (`get_due_cards` +
  threshold check + sound).
- Personal `CARDS_REPO_URL` lives in this script; if bsrc ever needs
  to be sharable, move it to `~/.config/bsrc/anki.local.env`.
